# FB-001 — TASK-001 Teslimatı Düzeltme Talebi

Teslimat denetlendi (rapor: `docs/reviews/REVIEW-2026-06-11-TASK-001.md`).
**Uygulama import aşamasında çöküyor; testler hiç koşamıyor.** Aşağıdaki bulguları düzelt ve
SADECE değişen dosyaları `### FILE:` blok formatında yeniden teslim et.

## Bulgu 1 — `Optional` import eksik (CRITICAL)
- **Dosyalar:** `app/models/audit.py`, `app/models/base.py`
- **Reproduce:** `pytest tests` → `NameError: name 'Optional' is not defined` (audit.py:25)
- **Beklenen:** `from typing import Optional` eklenmeli (her iki dosyaya).

## Bulgu 2 — `RefreshToken` modelinde geçersiz tip ifadesi (CRITICAL)
- **Dosya:** `app/models/user.py`
- **Sorun:** `BaseModel.__annotations__["id"].type` — `Mapped[uuid.UUID]` annotation'ının `.type` özelliği yoktur; sınıf tanımı anında `AttributeError` fırlatır.
- **Beklenen:** Doğrudan `UUID(as_uuid=True)` kullan (`from sqlalchemy.dialects.postgresql import UUID`) — ama Bulgu 5'teki taşınabilirlik notuna da uy.

## Bulgu 3 — Döngüsel import (CRITICAL)
- **Dosya:** `app/core/audit.py` satır: `from app.core.auth import get_current_user`
- **Zincir:** main → core.audit → core.auth → models.user → models.base → core.audit (yarım) → ImportError
- **Beklenen:** Bu üst-seviye import kaldırılmalı (zaten kullanılmıyor; dispatch içinde lokal import var).

## Bulgu 4 — requirements.txt yanlış JWT paketi (CRITICAL)
- **Sorun:** Kod `import jwt` (PyJWT) kullanıyor; requirements'ta `python-jose` var, `PyJWT` yok → ImportError.
- **Ayrıca:** `aioredis==2.0.1` Python 3.11'de import hatası verir ve kodda hiç kullanılmıyor.
- **Beklenen:** `PyJWT==2.8.0` ekle; `python-jose` ve `aioredis` satırlarını kaldır. Test için `aiosqlite` ekle.

## Bulgu 5 — Testler SQLite'ta koşamaz: PostgreSQL-özel tipler (HIGH)
- **Dosya:** `app/models/base.py`, `tests/conftest.py`
- **Sorun:** `UUID(as_uuid=True)` (postgresql dialect) + `server_default=text("gen_random_uuid()")` SQLite test DB'sinde çalışmaz.
- **Beklenen:** SQLAlchemy 2.0'ın dialect-bağımsız `sa.Uuid` tipini kullan; `gen_random_uuid()` server_default'unu kaldır (Python tarafı `default=uuid.uuid4` yeterli) ya da migration'da PostgreSQL'e özgü bırak, modelde bırakma.

## Bulgu 6 — pytest-asyncio yapılandırması eksik (HIGH)
- **Sorun:** Async fixture'lar `@pytest.fixture` ile tanımlı; strict modda await edilmez → testler bozuk.
- **Beklenen:** `backend/pytest.ini` ekle:
  ```ini
  [pytest]
  asyncio_mode = auto
  ```
  ya da tüm async fixture'ları `@pytest_asyncio.fixture` yap.

## Bulgu 7 — Config zorunlu alanlar testte patlıyor (HIGH)
- **Dosya:** `app/core/config.py`
- **Sorun:** `DATABASE_URL` ve `JWT_SECRET_KEY` zorunlu (`Field(...)`); `.env` yoksa import anında ValidationError.
- **Beklenen:** Test-dostu güvenli default'lar ver (örn. `sqlite+aiosqlite:///./dev.db`, `dev-secret-change-me`) — production'da `.env` zaten override eder. `tests/conftest.py` başında da env değişkenlerini set et.

## Bulgu 8 — Zorunlu model alanları eksik (MEDIUM)
- **Dosya:** `app/models/user.py` (`RefreshToken`)
- **Sorun:** 02 Talimatları §0.3: her tablo `created_at, updated_at, deleted_at...` içermeli. `refresh_tokens` tablosunda `updated_at/deleted_at/created_by/updated_by` yok.
- **Beklenen:** `RefreshToken` da `BaseModel`'den türesin. (`audit_logs` append-only istisnası kabul — değiştirme.)
- Migration dosyası da buna göre güncellenmeli.

## Bulgu 9 — `__init__.py` dosyaları eksik (LOW)
- **Beklenen:** `app/`, `app/core/`, `app/models/`, `app/routers/`, `app/schemas/`, `tests/` altına boş `__init__.py` ekle.

## Bulgu 10 — alembic.ini konumu (LOW)
- **Sorun:** `migrations/alembic.ini` içinde `script_location = migrations` → backend kökünden `alembic upgrade head` çalışmaz.
- **Beklenen:** `alembic.ini` `backend/` köküne taşınmalı.

## Kabul Koşulu (2. tur)
- `DATABASE_URL` ve `JWT_SECRET_KEY` env'siz, sadece `pip install -r requirements.txt && pytest` ile **tüm testler yeşil**.
- `docker compose up` + `alembic upgrade head` + `python seed.py` akışı PostgreSQL'de çalışır durumda.
- Sadece değişen dosyaları gönder; değişmeyenleri tekrar üretme.
